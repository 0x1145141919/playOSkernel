# FreePagesAllocator::free_pages_in_seg_control_block (BCB) 文档

## 1. 引言

`free_pages_in_seg_control_block`（简称 BCB）是 FreePagesAllocator 中的核心类，用于实现一个基于位图（bitmap）的二进制伙伴分配器（binary buddy allocator）。它管理一段连续的物理内存段（segment），以 4KB 页为最小单位，支持 power-of-2 大小的块分配和释放。BCB 设计用于内核级内存管理，支持最大 64 阶（order），但实际使用中通常限制在 20~30 阶以避免过大内存开销。

BCB 的主要目标是高效管理物理页的分配和释放，减少碎片化，通过位图编码空闲状态，并使用提示（hint）和间隔查找优化性能。整个系统假设物理地址（phyaddr_t）至少 4KB 对齐，但不要求对最大 order 对齐。

本文档基于提供的源文件（FreePagesAllocator_BCB.cpp、FreePagesAllocator.h、mixed_bitmap.cpp 和 BCB_test.cpp）讲解 BCB 的核心数据结构、用途、编码方式以及核心算法。

## 2. 核心数据结构及其用途

BCB 使用紧凑的数据结构来跟踪内存状态，避免链表开销。核心结构围绕位图和索引构建，支持快速查找和更新。

### 2.1 mixed_bitmap_t
- **定义位置**：FreePagesAllocator.h 和 mixed_bitmap.cpp。
- **用途**：这是一个继承自 `bitmap_t` 的位图类，用于管理位（bit）级状态。BCB 中只有一个全局 `mixed_bitmap_t` 实例（`order_freepage_existency_bitmaps`），它统一管理所有 order 的空闲状态，而不是每个 order 一个位图。这提高了空间效率（总位图大小为 2^{MAX_SUPPORT_ORDER + 1} 位）。
- **关键成员**：
  - `uint64_t entry_count`：位图总条目数（位数）。
  - `uint64_t* bitmap`：底层 uint64_t 数组存储位图数据。对于第一个 BCB，使用固定数组 `FIRST_BCB_FREE_PAGES_FIXED_BITMAP` 以避免动态分配。
- **关键方法**：
  - `bit_set(idx, value)`：设置位为 true（1）或 false（0）。
  - `bit_get(idx)`：获取位值。
  - `find_free_in_interval(start_idx, interval_length)`：在指定区间内查找第一个为 1 的位（空闲位）。返回偏移或 ~0ULL（未找到）。实现使用逐位检查 + __builtin_ctz 优化 64 位块扫描。
  - `second_stage_init()`：初始化位图，所有位设为 0；对于第一个 BCB，使用预分配数组。

### 2.2 order_freepage_existency_bitmaps
- **用途**：单一 `mixed_bitmap_t` 实例，编码所有 order 的空闲块状态。位图按 order 分段存储（通过 `order_bases` 分隔）。
- **编码方式**：见第 3 节。

### 2.3 uint64_t order_bases[DESINGED_MAX_SUPPORT_ORDER]
- **用途**：每个 order 在全局位图中的起始偏移（base）。允许将单一位图逻辑分为 per-order 段。
- **初始化**：在 `second_stage_init()` 中计算：
  - `order_bases[0] = 0`
  - `order_bases[order+1] = order_bases[order] + (1ULL << (MAX_SUPPORT_ORDER - order))`
  - 这确保 order k 有 2^{MAX_SUPPORT_ORDER - k} 个位，匹配二叉树结构（高 order 位少，低 order 位多）。
- **示例**（MAX_SUPPORT_ORDER=20）：
  - order 0: 基址 0，长度 2^20 位（1M 位）。
  - order 1: 基址 2^20，长度 2^19 位。
  - ... 
  - order 20: 基址 ≈ 2^21 - 1，长度 1 位。

### 2.4 uint64_t suggest_order_free_page_index[DESINGED_MAX_SUPPORT_ORDER]
- **用途**：每个 order 的分配提示（hint）。记录上一个空闲块的索引，用于 first-fit 优化。下次分配从这里开始检查，避免从 0 扫描。
- **初始化**：全设为 ~0ULL（INVALID_INBCB_INDEX）。
- **更新**：在 `free_page_without_merge()` 中更新为新释放的索引。在分配时，如果 hint 有效且位为 1，则直接使用；否则 fallback 到全扫描。

### 2.5 struct BCB_statistics
- **用途**：收集运行时统计，用于调试和性能分析。
- **成员**：
  - `free_count[order]`：每个 order 的空闲块数。
  - `suggest_hit/miss[order]`：hint 命中/未命中计数。
  - `scan_count`：全扫描次数。
  - `fold_count_success/fail`：合并成功/失败次数。
  - `split_count`：分裂次数。
- **用途**：在分配/释放/分裂/合并时更新，帮助诊断碎片化或效率问题。

### 2.6 其他辅助成员
- `phyaddr_t base`：内存段起始物理地址（4KB 对齐）。
- `uint8_t MAX_SUPPORT_ORDER`：最大 order（构造函数参数，通常 20）。
- `trylock_cpp_t lock`：用于并发保护（保证一个BCB最多只能被一个CPU使用，freepagesallocator层面的多BCB体系）。
- `bool is_splited_bitmap_valid`：标志位图是否有效（初始化后设为 true）。

## 3. 编码方式

BCB 使用位图编码内存状态，强调“存在性”（existency）：
- **位值含义**：
  - 1（true）：该索引对应的块是空闲的（free page exists）。
  - 0（false）：该块已分配、分裂或不存在。
- **索引计算**：
  - 对于 order k，位图偏移 = `order_bases[k] + idx`，其中 idx 是该 order 下的逻辑块索引（0 ~ 2^{MAX_SUPPORT_ORDER - k} - 1）。
  - 物理地址 = `base + (idx << (order + 12))`（因为 order 0 = 4KB = 2^12 字节）。
- **二叉树隐式编码**：
  - 位图模拟伙伴系统的二叉树：order k 的 idx 对应 order k-1 的 2*idx 和 2*idx+1。
  - 合并/分裂时，通过位操作（如 idx ^ 1 计算伙伴，idx >> 1 计算父）维护树一致性。
- **初始化编码**：
  - 初始时，整个段视为一个 order MAX_SUPPORT_ORDER 的空闲块：只设 `order_bases[MAX_SUPPORT_ORDER] + 0 = 1`，其他位为 0。
- **一致性**：释放时检查二叉树一致性（e.g., 父位不应已为 1），若违反则 fatal error。

这种编码方式空间高效（位图总大小 ≈ 2^{MAX+1} 位 ≈ 2 * 总页数位），时间上依赖位操作和内置函数（如 __builtin_ctzll、__builtin_clzll）优化。

## 4. 核心算法

BCB 实现标准伙伴系统算法：分配时向上搜索 + 向下分裂；释放时标记 + 向上合并。支持 top-down 折叠以主动去碎片。

### 4.1 size_to_order(uint64_t size)
- **算法**：计算最小 order k，使得 2^{k} * 4KB >= size（向上取整）。
- **步骤**：
  1. num_pages = ceil(size / 4096)
  2. next_pow2 = 使用位操作计算 num_pages 的下一个 2 的幂。
  3. order = log2(next_pow2) = 63 - __builtin_clzll(next_pow2)
- **用途**：标准化请求大小。

### 4.2 allocate_buddy_way(uint64_t size, KURD_t& result)
- **算法**：从目标 order 向上搜索空闲块，若找到高 order 块则分裂。
- **步骤**：
  1. order = size_to_order(size)
  2. 对于 i = order 到 MAX_SUPPORT_ORDER：
     - 先检查 hint (`suggest_order_free_page_index[i]`) 是否为空闲位（1）。
     - 若是：若 i > order，则 split_page(hint_idx, i, order)；标记位为 0；更新统计；返回地址。
     - 若 hint 失效：全扫描 `find_free_in_interval(order_bases[i], 2^{MAX-i})`，找到第一个 1，返回偏移作为 idx。
  3. 未找到：返回 error (NO_AVAILABLE_BUDDY)。
- **优化**：hint 减少扫描；间隔查找支持局部性。

### 4.3 split_page(uint64_t splited_idx, uint8_t splited_order, uint8_t target_order)
- **算法**：递归向下分裂块，直到目标 order。
- **步骤**：
  1. 若 splited_order == target_order：标记位为 0，返回。
  2. 检查 splited_order 位是否为 1（空闲），否则 error。
  3. 标记 splited_order 位为 0。
  4. 设置左子块 (2*splited_idx) 为 1（空闲）。
  5. 调用 free_page_without_merge(2*splited_idx + 1, splited_order-1) 设置右子块为 1 并更新 hint。
  6. 递归 split_page(2*splited_idx, splited_order-1, target_order)。
- **注意**：分裂后，只有一个子块用于进一步分裂，另一个保持空闲。

### 4.4 conanico_free(uint64_t in_bcb_idx, uint8_t order)
- **算法**：释放并向上合并（coalesce）。
- **步骤**：
  1. 检查参数有效性（order <= MAX, idx < 2^{MAX-order}, 位不为 1 以防 double free）。
  2. 设置位为 1（空闲）。
  3. current_idx = idx, current_order = order
  4. while current_order < MAX：
     - buddy_idx = current_idx ^ 1
     - 若 buddy 位 != 1：break
     - 清除两个子位为 0。
     - current_idx = current_idx >> 1
     - 检查父位 != 1（一致性），设置父位为 1。
  5. 更新统计。
- **用途**：核心释放逻辑，确保最大合并。

### 4.5 is_reclusive_fold_success(uint64_t idx, uint8_t order)
- **算法**：递归向下折叠（fold）二叉树，用于主动合并碎片。
- **步骤**：
  1. 检查左右子块 (2*idx, 2*idx+1) 是否均为 1。
  2. 若是：清除子位，设置父位为 1，返回 true。
  3. 否则：递归左右子树，若两者均成功：合并父，返回 true；否则 false。
- **用途**：在 `top_fold()` 中从根 (idx=0, order=MAX) 调用，实现全局去碎片。

### 4.6 free_buddy_way(phyaddr_t base, uint64_t size)
- **算法**：用户释放入口。计算 order 和 idx = (base - this->base) >> (order + 12)，调用 conanico_free。

### 4.7 second_stage_init()
- **算法**：分配位图，计算 order_bases，初始释放根块 (free_page_without_merge(0, MAX))。

### 4.8 辅助算法
- `free_page_without_merge(idx, order)`：简单设位为 1，更新 hint 和 free_count（不合并）。
- `top_fold()`：调用 is_reclusive_fold_success(0, MAX_SUPPORT_ORDER)。
- 打印方法（如 print_bitmap_order_interval_compress）：压缩打印位图区间，用于调试。

## 5. 测试和验证

BCB_test.cpp 提供单元测试：
- 创建 BCB (base=0x100000000, order=20)。
- 初始化并打印基本信息。
- 生成 10000 个随机大小 mem_seg。
- 运行 1000000 次状态机循环：随机分配/释放，模拟高负载。
- 打印最终信息，检查无崩溃/错误。

测试使用对数正态分布模拟真实负载，验证分配/释放一致性。
